##使用github搭建基于hexo框架的个人博客网站

###注册github账号，创建仓库

###下载hexo框架：
1. hexo框架是基于nodejs(js运行时环境)实现的，因此需要先安装nodejs
在nodejs官网下载nodejs安装包（msi），选择长期支持版本（比较稳定）
2. 配置npm全局安装路径：npm是nodejs的包管理器，我们需要使用它下载hexo框架相关包，配置好统一安装路径便于管理。npm set prefix “D:\NodeJs”(路径根据实际情况填写)
3. 更换镜像源：配置国内镜像源，下载更快捷。
npm config set registry https://registry.npmmirror.com
检查：npm get registry
4.安装hexo: npm install -g hexo-cli
 
###搭建hexo项目
1. 打开命令行界面（管理员执行），cd到想要保存项目的文件夹
2. 依次执行
hexo init hexo-blog 使用hexo框架创建一个项目
cd hexo-blog
npm install 该指令根据项目下的json文件自动安装相关依赖
3. 更换主题
cd hexo-blog
git clone -b master https://github.com/jerryc127/hexo-theme-butterfly.git themes/butterfly
然后在config的yml文件里将主题改名
如果后续第四项无法正常访问，建议先不更改主题
4. 启动
hexo generate 将source目录下的资源生成静态html文件(public文件夹)
hexo server 启动
访问localhost:4000可查看界面

###部署项目
部署到github上
使用git将项目上传到github
配置ssh
ssh，简单来讲，就是一个秘钥，其中，id_rsa是你这台电脑的私人秘钥，不能给别人看的，id_rsa.pub是公共秘钥，可以随便给别人看。把这个公钥放在GitHub上，这样当你链接GitHub自己的账户时，它就会根据公钥匹配你的私钥，当能够相互匹配时，才能够顺利的通过git上传你的文件到GitHub上。
1
git config –global user.name “yourname”
git config –global user.email “youremail”
配置为你的github用户名，邮箱
2
ssh-keygen -t rsa -C “youremail”
生成密钥
将密钥放入github相关settings中
cat ~/.ssh/id_rsa.pub
在GitHub的settings中新建一个ssh，把上述文件内容复制到其中
3
修改config的yml文件
deploy:
type: git
repository: https://github.com/gaoziman/gaoziman.github.io.git
branch: master
4.
下载部署工具
npm install hexo-deployer-git –save
5 部署
hexo clean
hexo generate
hexo deploy
6访问
用户名.github.io
 
###写文章
hexo new post “我的第一篇博客”
在source的post目录下找到对应md文件，编辑，重新部署即可
